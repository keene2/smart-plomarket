# 产品需求文档 (PRD) — Smart Polymarket v1

## 1. Context：为什么做 & 解决什么问题

Polymarket 是全球最大的链上预测市场，但它的原生产品只解决"下注"本身，**不解决"跟谁下注"**：

- 谁是这个市场的大玩家？他们持仓方向如何？
- 有没有历史胜率稳定、赚钱能力强的地址值得跟？
- 事件型 KOL（记者、政客分析师、体育玩家）在哪个市场下了多大注？

GMGN 在 Solana memecoin 场景已经验证："**聚合行情 + 聪明钱画像**"这个数据层比交易所本身更有粘性。本项目把同一套心智搬到预测市场：

- **用户熟悉环境**：中文为主（团队有 GMGN 背景，用户群重合）
- **差异化护城河**：聪明钱 / 鲸鱼 / KOL 监控面板，而不是另一个 Polymarket 壳
- **上手零门槛**：Privy 提供 TEE/MPC 嵌入式钱包，邮箱/Twitter 登录即可下单
- **不与 Polymarket 竞争流动性**：结算与撮合沿用 Polymarket（Polygon + USDC），我们只做聚合层

MVP 聚焦"**能看行情、能下单、能看聪明钱**"三件事。跟单（copy-trade）留到 v2。

## 2. 核心用户画像

| 用户 | 场景 | 关键痛点 |
| --- | --- | --- |
| 中文 Web3 散户（主力） | 想下注但不会用 Polymarket 原生钱包 / 不知道谁靠谱 | 语言 + 钱包门槛 + 信息不对称 |
| 从 GMGN 过来的交易者 | 把"跟聪明钱"的习惯迁移到新赛道 | 缺一个和 GMGN 同风格的界面 |
| 事件型 KOL / 内容账号 | 希望展示自己的 Polymarket 战绩 | 原生 Polymarket 没有公开画像页 |

## 3. V1（MVP）功能范围

### 3.1 账户与钱包（必须，最先做）

- **登录方式**：Privy 托管的邮箱 / Google / Twitter / WalletConnect（外部钱包保留通道）
- **嵌入式钱包**：Privy 的 TEE 钱包，用户首次登录自动创建，私钥不落到服务器
- **充值**：展示钱包地址 + 二维码，提示从 CEX / 桥接充 USDC.e（Polygon）
- **提现**：转账到用户指定外部地址
- **安全**：Privy 默认 MFA / 邮箱确认；设置页提供私钥导出（自托管出口）
- **不做**：法币入金、跨链聚合桥（v2）

### 3.2 行情 / 市场浏览

- **首页**
  - 热门市场卡片（按 24h 成交额 / 活跃度）
  - "聪明钱正在下注"实时信息流
  - PnL 榜单 Top 10 入口
- **市场列表**
  - 分类（政治 / 加密 / 体育 / 科技 / 娱乐…，沿用 Polymarket tag）
  - 排序（热度 / 成交额 / 即将结算 / 新上线）
  - 搜索
- **市场详情页（关键页面）**
  - 概率走势图
  - 简介 + UMA 结算规则 + 结束时间
  - 订单簿（YES / NO 两侧）
  - 最近成交列表
  - **差异化模块**：该市场内聪明钱持仓分布 + 鲸鱼近期动作（必须放在首屏）
- **下单**
  - Buy YES / Buy NO，市价 / 限价
  - 滑点提示 + gas 预估
  - Polymarket CLOB REST API 提交（EIP-712 签名由 Privy 钱包完成）

### 3.3 聪明钱监控（差异化核心）

优先级：**大额持仓/下注鲸鱼 > 身份已知 KOL > 结算后 PnL/ROI 榜**

#### 3.3.1 鲸鱼实时流（P0）

- 全局 feed：按成交金额 > 阈值（默认 10,000 USDC，可调）实时推送
- 按市场筛选
- 字段：时间、地址（带标签）、市场、方向、金额、价格、持仓变化
- 实时性：WebSocket，延迟 < 15s

#### 3.3.2 KOL / 已标注地址（P0）

- 预置标签库：运营 + 社区投稿
- KOL 页：按分类查看已标注地址及其近期动作
- 关注机制：follow 任意地址，Portfolio 页的 watchlist
- 通知：站内消息 + Email；TG bot 放 v1.5

#### 3.3.3 PnL / ROI 榜单（P1）

- 维度：24h / 7d / 30d / All time
- 指标：已实现 PnL、未实现 PnL、总成交额、胜率、交易次数
- 过滤：最小交易次数 ≥ N（避免一次大赢的运气型）
- 默认排序：综合分 = f(PnL, 胜率, 活跃度)，详见 [whale-algorithm.md](whale-algorithm.md)

#### 3.3.4 地址详情页（任意地址可点开）

- Header：地址、标签（若有）、累计 PnL、胜率
- 当前持仓（按市场聚合）
- 历史交易流水（可按市场过滤）
- 盈亏曲线
- 关注按钮

### 3.4 个人 Portfolio

- 我的持仓（按市场聚合，当前市值 / 成本 / 未实现盈亏）
- 历史交易
- Watchlist（关注的聪明钱）
- 通知中心

### 3.5 通知与分享

- 站内通知：关注地址新交易、订单成交、市场结算
- Email：关键事件
- **分享卡片（中文增长抓手）**：分享持仓 / 市场 / 聪明钱地址，生成带水印图片，适配 Twitter / 小红书 / 微信

### 3.6 国际化

- 中文优先；UI 文案、市场标题、分类名人工校对
- i18n 框架预留英文槽位
- 市场标题用 LLM 自动中译兜底，高热度市场人工审校

## 4. 明确不在 V1 范围

- ❌ 一键跟单 / 自动 copy-trade（v2 核心）
- ❌ 自营撮合 / 自建 AMM
- ❌ 法币入金、跨链桥
- ❌ Kalshi / Limitless 等非 Polymarket 数据源
- ❌ 移动 App（v1 只做响应式 Web / PWA）
- ❌ 自建 TEE 签名服务
- ❌ 跨链地址画像（v2 差异化升级点）

## 5. 开放问题（待决策）

1. **PnL 口径**：已实现 vs. 含未实现（mark-to-market）？建议两者都展示，榜单默认综合分。
2. **鲸鱼阈值默认值**：10k / 25k / 50k USDC？建议分层（普通 1k / 鲸鱼 10k / 超级鲸鱼 50k）。
3. **KOL 冷启动数量**：运营 seed 多少？建议首批 50 个（政治 20 + 加密 20 + 体育 10）。
4. **合规免责**：产品定位为数据层，下单发生在 Polymarket；ToS 与首页需明确说明。
5. **商业模式**：v1 建议不开启，v2 走"跟单抽成 + 高级数据订阅"。v1 先埋点。
6. **数据源托底**：Subgraph / CLOB 偶尔会挂。MVP 单活 + 报警，v1.5 加自建 Polygon archive node。

## 6. 验证计划（MVP 上线后的成功标准）

- **北极星指标**：周活跃"看聪明钱 → 下单"转化用户数
- **功能验证**
  - 用户能在 Privy 登录后 3 分钟内完成首单
  - 鲸鱼 feed 延迟 < 15s
  - PnL 榜数字与 Dune / Polymarket 原站偏差 < 2%
- **用户测试**：10 位中文 Polymarket 老用户 + 10 位未用过的 GMGN 用户做可用性测试
- **数据对账**：每日对 10 个随机地址做 trade count 对账（对比 Subgraph 原始数据）
